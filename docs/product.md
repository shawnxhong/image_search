# Product Overview

## What It Does

Agentic Image Search is a desktop app for searching and organizing a personal photo library. All AI inference runs locally via OpenVINO — no cloud APIs, no data leaves the machine.

Users ingest images (extracting metadata, captions, and faces), label detected faces, and then search across their library using natural language queries. An LLM agent interprets each query and decides which combination of search tools to use.

## Core Workflows

### 1. Ingestion

The user selects image files via a native OS file dialog. Each image goes through an automated pipeline:

1. **EXIF extraction** — capture timestamp, GPS coordinates
2. **Reverse geocoding** — GPS coordinates resolved to country/state/city
3. **VLM captioning** — one-sentence visual description generated by Qwen2.5-VL
4. **Face detection** — bounding boxes and identity descriptors extracted
5. **Face matching** — detected faces compared against known identities, top candidates suggested
6. **Embedding** — caption text encoded into a 384-dim semantic vector for search

After ingestion, if faces are detected, the user is prompted to label them:
- **Candidate suggestions** — clickable buttons showing previously known people with similarity scores
- **Free text input** — for naming new people
- **Dismiss** — mark false positive detections as "not a face"

Once all faces are named or dismissed, the caption is automatically **refined** with person names included (e.g. "Two people at the beach" becomes "Alice and Bob at the beach"), making future searches more specific.

### 2. Search

The user types a natural language query like "photos of Alice at the beach last year". A Qwen3-4B LLM agent reads the query and decides which tools to invoke:

- **Caption search** — semantic similarity over VLM-generated captions
- **Person search** — partial name matching against labeled faces
- **Time search** — natural language time expressions ("last year", "January", "yesterday")
- **Location search** — country, state, or city substring matching

The agent can call multiple tools and loop to refine its approach. Results are assembled into two lists:

- **Solid matches** — images returned by ALL tools called (intersection). High confidence.
- **Soft matches** — images returned by SOME tools but not all. Partial relevance.

Both lists are sorted by semantic similarity score. The user sees the agent's reasoning steps in real time via streaming.

### 3. Library Browsing

The Library tab shows all ingested images sorted by date (most recent first). Each image card displays:
- Capture date
- Location (city, state, country — from reverse geocoding)
- VLM-generated caption
- File path

The library uses cursor-based pagination (50 images per page) with a "Load More" button. No models need to be loaded — this is a pure database view.

### 4. Model Management

Since the app runs on a PC with limited hardware, not all AI models can be loaded simultaneously. The Model Control Panel lets users load and unload four model groups:

| Model | Used For |
|-------|---------|
| LLM (Qwen3-4B) | Agent reasoning during search |
| VL Captioner (Qwen2.5-VL) | Image captioning during ingestion |
| Embeddings (all-MiniLM-L6-v2) | Text encoding for both search and ingestion |
| Face Detection (Intel OMZ) | Face detection during ingestion |

Two preset buttons handle the common configurations:
- **Search Mode** — loads LLM + Embeddings, unloads VLM + Face Detection
- **Ingest Mode** — loads VLM + Embeddings + Face Detection, unloads LLM

The app validates that required models are loaded before allowing search or ingestion, showing a warning if not.

## Query Modes

| Mode | Input | Description |
|------|-------|-------------|
| Text | Natural language query | Agent-driven multi-tool search |
| Image | Image file path | Similar-image retrieval via embedding |
| Image + Text | Both | Image similarity with additional text constraints |

## Search Result Contract

Every search returns a `DualListSearchResponse`:

- **Solid results** — passed all tool filters, ranked by semantic score
- **Soft results** — partial matches, deduplicated against solid list

Each result includes:
- Image thumbnail
- Similarity score (0–1)
- File path
- Match explanation (which tools matched and why)

## Ingestion Status Lifecycle

```
pending → processing → ready
                     → pending_labels → (user labels faces) → ready
                     → failed
```

An image becomes fully searchable at `ready` status. Images at `pending_labels` are searchable but may have generic captions that don't include person names.
